# AGENTS.md — How agents operate this repo

This is the permanent, repo-level contract for every AI agent working in AI Topic Scout: **Claude Code, Codex CLI, GitHub Copilot (GHCP), Copilot CLI, Microsoft scouting-style agents, Claw, swarm workers,** and human operators.

`AGENTS.md` explains the reusable tool surface. `TOPIC_AGENTS.md` (generated by `make init`) defines the *active* research topic and must be read before topic-specific work.

---

## 1. Repo purpose

Use this repo when the goal is to:

- turn a raw research intent into a structured topic contract (`topic.json`)
- discover papers via OpenAlex + citation-graph expansion
- rank candidate papers with an LLM
- maintain an accepted paper corpus (`data/papers.json`)
- produce a synthesis report, opportunity analysis, and interactive dashboard
- emit multi-agent task manifests for other runtimes

Treat the repo as a **workspace generator first, research workspace second**. If `topic.json` is missing, the first step is always: initialize or ask for the intent needed to initialize. If it exists, do not overwrite unless the user asks to reset.

## 2. Command surface

```bash
make init            # refine intent → topic.json + agents/ + skills/ + TOPIC_AGENTS.md
make scout           # OpenAlex + LLM ranking → data/candidates.json
make review          # print candidate review queue
python3 scripts/accept_candidates.py <id>...   # accept into corpus
make corpus          # notes + reports/research_report.md
make opportunities   # data/research_opportunities.json
make opportunities-check   # validate opportunity JSON
make dashboard       # topic-dashboard.html + data/dashboard.json
make reset           # wipe generated workspace
make test            # unit tests

python3 scripts/orchestrate.py plan                          # show task plan
python3 scripts/orchestrate.py run --mode sequential         # local runner
python3 scripts/orchestrate.py emit --mode claw              # data/claw_tasks.json
python3 scripts/orchestrate.py emit --mode swarm             # data/swarm_tasks.json
python3 scripts/orchestrate.py emit --mode copilot           # data/copilot_tasks.json
python3 scripts/orchestrate.py emit --mode copilot-cli       # data/copilot-cli_tasks.json
python3 scripts/orchestrate.py emit --mode microsoft-scouting
```

Non-interactive init:

```bash
python3 scripts/init_topic.py --intent "<intent>"
python3 scripts/init_topic.py --topic ... --goal ... --audience ... \
    --include ... --exclude ... --years 2023-2026 --taxonomy ...
python3 scripts/init_topic.py --offline    # no LLM refinement
```

## 3. Universal workflow (every agent follows this)

1. **Inspect state.** Check `topic.json`, `TOPIC_AGENTS.md`, `data/papers.json`, `data/candidates.json` before mutating anything. Read only the role brief under `agents/` relevant to the current task.
2. **Contract.** Run `make init` (or `init_topic.py` with flags) to produce `topic.json`. Confirm it contains topic, goal, audience, include/exclude, years, taxonomy, dashboard sections, search queries.
3. **Discover.** `make scout` (or `QUERY="…" make scout` for a targeted supplement). Preserve OpenAlex IDs, DOIs, URLs, abstracts, citation counts, provenance, scores, cost metadata. `data/candidates.json` is a *queue*, not a corpus.
4. **Review.** `make review`. Accept only candidates that satisfy `topic.json`. Reject adjacent-but-out-of-scope. If human approval is required, stop at recommendations. Accept with `scripts/accept_candidates.py <ids>`.
5. **Curate corpus.** `data/papers.json` is the corpus of record. Deduplicate by OpenAlex ID / DOI / arXiv ID / title / URL. Never invent metadata. Preserve scout history + cost metadata.
6. **Rebuild synthesis** *only after accepted corpus changes*: `make corpus` → `make opportunities` → `make opportunities-check` → `make dashboard`. If no new paper accepted, do not regenerate reports just to show activity.
7. **Emit manifests** when another runtime executes the work. Each manifest ships topic contract + commands + role brief path + inputs/outputs + dependencies + required contract files.
8. **Validate before claiming success.** Run the command tied to your change. Run `make test` after code changes. Confirm JSON parses. Inspect generated Markdown/HTML for substance, not just presence.

## 4. Per-agent instructions

### 4.1 Claude Code

- Claude auto-loads `AGENTS.md`. Confirm you've read it and `TOPIC_AGENTS.md` (if present) before acting.
- Prefer `make` targets over ad-hoc Python. Run commands directly.
- When the user says *"scout papers on X"*: if `topic.json` exists and matches X, skip init; otherwise `make reset && make init` (ask before reset).
- After edits: `make test`, then run the affected pipeline step end-to-end.
- Do not fabricate paper metadata, abstracts, or citations. If OpenAlex or the LLM fails, surface the failure — do not paper over it.
- Use the Read/Edit/Write tools for `topic.json` and generated artifacts; use Bash for `make`, `scripts/*.py`, and `git`.

### 4.2 Codex CLI

- Launch `codex` in the repo root. `codex login` first if not already authed — `make init` uses Codex CLI by default (no API key needed).
- Primary verification surface: `make init`, `make scout`, `make corpus`, `make opportunities`, `make dashboard`, `make test`.
- Keep edits scoped. When preparing work for other agents, emit the appropriate manifest and inspect it for completeness before handing off.

### 4.3 GitHub Copilot (GHCP)

- Start from `data/copilot_tasks.json`. If missing, run `make init` then `python3 scripts/orchestrate.py emit --mode copilot`.
- Follow the task **dependency order**. Read the `contract_files` listed on each task, then read the `agent_brief` for the assigned role under `agents/`.
- Modify only the outputs declared by the current task. Report any required human approval before corpus ingestion.

### 4.4 Copilot CLI

- Start from `data/copilot-cli_tasks.json`. If missing, run `make init` then `python3 scripts/orchestrate.py emit --mode copilot-cli`.
- Execute **one manifest task at a time**. Run the `command` field when applicable.
- After each task, print concise status: changed files, accepted IDs, rejected IDs, validation results.
- Do not run dashboard/report regeneration until the dependency chain reaches the publish step.

### 4.5 Microsoft scouting-style agents

- Start from `data/microsoft-scouting_tasks.json`. If missing, run `make init` then `python3 scripts/orchestrate.py emit --mode microsoft-scouting`.
- Treat the manifest as a **research operations plan**: query design, graph discovery, relevance review, library maintenance, synthesis, gap analysis, and publishing are separate responsibilities.
- Preserve evidence provenance. Make uncertainty explicit in review notes and opportunity analysis.

### 4.6 Claw-style agents

- Start from `data/claw_tasks.json`. If missing, emit `--mode claw` after `make init`.
- A **coordinator** assigns each task to the matching role brief in `agents/`. Workers write only their listed outputs. The coordinator gates final acceptance and publication.

### 4.7 Swarm agents

- Start from `data/swarm_tasks.json`. If missing, emit `--mode swarm` after `make init`.
- Workers may act in parallel **only** where manifest dependencies allow it. Order: query design → discovery → review → corpus → (synthesis, opportunities, dashboard).

### 4.8 Human operators

- Safest manual path: `make init` → `make scout` → `make review` → `scripts/accept_candidates.py <ids>` → `make corpus` → `make opportunities` → `make opportunities-check` → `make dashboard`.

## 5. Role responsibilities

| Role | Responsibility |
|---|---|
| `coordinator` | Enforce `topic.json` + `TOPIC_AGENTS.md`; assign work; check dependencies; gate publication. |
| `query_designer` | Improve `search_queries` without broadening excluded scope. Add synonyms, benchmarks, method names, adversarial queries. |
| `graph_scout` | Run scholarly discovery + citation-neighborhood expansion. Write candidates with provenance. Never accept into corpus. |
| `relevance_reviewer` | Apply include/exclude to `data/candidates.json`. Return accepted / rejected / uncertain IDs with reasons grounded in the contract. |
| `librarian` | Update `data/papers.json` only from approved IDs. Deduplicate. Preserve identifiers, URLs, and scout history. |
| `analyst` | Corpus synthesis. Classify papers into the taxonomy. Write notes + `reports/research_report.md`. |
| `gap_analyst` | Run opportunity analysis via `skills/analyze-research-gaps/SKILL.md`. Distinguish corpus gaps from field-wide gaps. Validate with `make opportunities-check`. |
| `publisher` | Regenerate `topic-dashboard.html` + `data/dashboard.json` **only** after accepted corpus or opportunity changes justify it. |

Full briefs: `agents/*.md`.

## 6. Hard rules

- Candidate discovery is **not** acceptance.
- Never invent metadata, abstracts, URLs, citations, methods, or conclusions.
- Every paper retains a source URL and stable identifier.
- Every opportunity states evidence, inference, uncertainty, and future queries.
- Distinguish corpus gaps from field-wide gaps.
- If no new paper is accepted, do not rewrite tracked artifacts.
- When scope is ambiguous, **tighten the contract** — do not broaden the corpus.
- When evidence is missing, record uncertainty and future scout queries — do not invent conclusions.

---

## 7. Failure handling

- **OpenAlex unavailable:** keep corpus unchanged; report the failed discovery step.
- **LLM ranking fails:** preserve OpenAlex candidates; either retry with another provider or mark the run as heuristic/offline.
- **Opportunity generation fails:** do not publish a dashboard implying fresh opportunity analysis.
- **JSON validation fails:** fix or regenerate before claiming success.

## 8. Repo rules

- Repository root is the live generated workspace surface.
- Do not assume the current topic — inspect `topic.json` + `TOPIC_AGENTS.md`.
- `AGENTS.md` (this file) is permanent. `TOPIC_AGENTS.md` is generated and topic-specific.
- Generated role briefs live under `agents/`.
- Generated topic-specific skills live under `skills/topic-paper-scout/` and `skills/analyze-research-gaps/`.
- `examples/` is illustrative, not the active workspace.
